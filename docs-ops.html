Wiki.js 維運手冊

oupaul/wiki · 自訂版（含剪貼簿貼圖功能）· 2026-06

原始碼：github.com/oupaul/wiki · 部署目錄：/opt/wikijs · PostgreSQL 17
一、一鍵部署
全新 Ubuntu 22.04 / 24.04，執行一行指令完成所有安裝
需要 root 權限 Ubuntu 22.04 / 24.04 首次約 15～20 分鐘
1
SSH 進入新主機，執行一鍵部署
腳本會自動安裝 Docker、Clone 原始碼、Build image、啟動服務
curl -fsSL https://raw.githubusercontent.com/oupaul/wiki/main/deploy.sh | sudo bash
2
腳本完成後，記錄輸出資訊
密碼為隨機產生，請立即保存
Wiki.js 網址   http://YOUR_IP
資料庫密碼     AbCdEfGhXxXx  ← 請妥善保存
設定目錄       /opt/wikijs
3
等待 30～60 秒後開啟網址，進行初次設定
查看啟動進度：
docker logs -f wiki

部署後的目錄結構

/opt/wikijs/
  ├── docker-compose.yml   ← 服務設定（含 DB 密碼）
  ├── src/                 ← Wiki.js 原始碼
  └── update.sh            ← 日後更新腳本
若要自訂 port（預設 80），可在執行前設定環境變數：export HOST_PORT=8080，再執行 curl 指令。
二、資料備份
備份 PostgreSQL 資料庫及所有上傳檔案
在舊主機或來源主機執行 需要 root 權限

備份範圍

項目內容儲存位置
PostgreSQL 資料庫所有頁面、使用者、權限、設定、留言、標籤Docker volume
上傳的媒體檔案圖片、文件等媒體庫內容/wiki/data/content
1
在來源主機執行備份腳本
curl -fsSL https://raw.githubusercontent.com/oupaul/wiki/main/migrate-export.sh | sudo bash
2
備份完成，確認備份檔
ls -lh /tmp/wikijs-backup-*.tar.gz
備份檔命名格式：wikijs-backup-YYYYMMDD-HHMMSS.tar.gz
3
將備份檔傳到新主機
在新主機上執行（將 OLD_IP 替換成來源主機 IP）
scp root@OLD_IP:/tmp/wikijs-backup-*.tar.gz /tmp/
建議定期備份（每週或每次重大更新前），將備份檔存至本機或雲端儲存。
三、資料還原
將備份還原至已完成一鍵部署的新主機
在新主機執行 需要先完成一鍵部署 需要 root 權限
還原會清空新主機現有的資料庫並覆蓋所有上傳檔案。請確認新主機是全新安裝，或資料可以被覆蓋。
1
確認備份檔已在新主機上
ls /tmp/wikijs-backup-*.tar.gz
2
下載並執行還原腳本
curl -fsSL https://raw.githubusercontent.com/oupaul/wiki/main/migrate-import.sh -o /tmp/migrate-import.sh

sudo bash /tmp/migrate-import.sh /tmp/wikijs-backup-YYYYMMDD-HHMMSS.tar.gz
將 YYYYMMDD-HHMMSS 替換成實際的備份檔名
3
還原完成後，確認服務正常
docker ps
docker logs wiki --tail=30

還原後必須確認的設定

  • Administration → General → 網站 URL（若 IP 或 Domain 有變更）
  • Administration → Storage → Git / S3 等外部 storage 連線
  • Administration → Authentication → OAuth / LDAP / SAML 設定
  • Administration → Mail → SMTP 寄信設定
四、版本更新
拉取最新程式碼並重新 Build image，Wiki.js 服務短暫中斷後自動恢復
在部署主機執行 重新 Build 約 5～15 分鐘
1
執行內建的更新腳本
自動完成：git pull → dev flag 關閉 → docker build → 重啟 wiki
sudo /opt/wikijs/update.sh
2
確認更新完成
docker ps
docker logs wiki --tail=20

手動更新（等效步驟）

cd /opt/wikijs/src
git pull

sed -i 's/"dev": true/"dev": false/' package.json
docker build -f dev/build/Dockerfile -t wikijs-custom:latest .
sed -i 's/"dev": false/"dev": true/' package.json

cd /opt/wikijs
docker compose up -d --no-deps wiki
更新前建議先執行備份，以防萬一需要回滾。
